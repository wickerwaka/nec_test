# v0.2 collision re-emission worklist

Trigger: the flat-1MB / collision-freedom work found ~5.22% of emitted v0.2 cases
have mod-64K footprint collisions (invalid on flat memory / upstream). Re-emission
plan, to run AFTER the main v0.2 emission completes and AFTER the board byte-identity
confinement check passes (FIRST post-emission spend).

## Two classes

1. INDEX-TARGETED (idx==seed holds; confined per-index re-emit via `emit_suite.py
   reemit --indices-file`): the ~86 affected forms with 0 logged rerolls. Only the
   colliding output-indices are re-emitted; attempt-0 preserves the original seed so
   non-colliding cases stay byte-identical.

2. FULL-FORM re-emit (idx != seed after a skip-to-next reroll in the running emission):
   **20, C7.0, 0F11, 7F** (each 1 "reserved page" ComposeError reroll). These must be
   fully re-emitted. Under the NEW confined per-index scheme (attempt-0 preserved,
   within-index reroll) this idx-shift class CANNOT recur.

## Gate
Merged files pass `_mirror_collision` scan at 0% AND schema spot-check. Regenerate the
per-form collision list post-emission (the running suite is still growing).
