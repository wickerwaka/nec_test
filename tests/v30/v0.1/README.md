# V30 (uPD70116) SingleStepTests-format suite — tranche v0.1

CPU test data captured from a real NEC uPD70116C-8 on the nec_test
harness (DE10-Nano, max mode, 4 MHz, zero wait states), in the
SingleStepTests V20 schema extended for the 16-bit bus per the 8086-suite
precedent. Generated by `sw/emit_suite.py` (seed base `v30-v0.1`; every
case is reproducible from `sha-derived seed f"{base}/{opcode}/{n}"` —
rerolled seeds logged in `emit_log.txt`).

This tranche is a 26-opcode / 500-cases-each sample: the emitter is the
product; full-scale emission is a Campaign 2 residual.

## Files

`XX.json.gz` / `XX.N.json.gz` (modrm-group) / `0FXX.json.gz` — one JSON
array of test objects per opcode form. `metadata.json` — per-opcode
status/flags-mask database (V20 masks are valid for the V30:
undefined-flag behavior is bit-exact, see
`docs/facts/undefined_flags.md`).

## Test object schema

Identical to the V20 suite (`docs/notes/singlesteptests_v20.md`):
`name`, `bytes`, `initial{regs,ram,queue}`, `final{regs,ram,queue}`,
`cycles`, `hash`, `idx`. Registers use Intel names (`ax..di, ip, flags`);
NEC aliases are listed in `metadata.json`. `final.regs` is sparse;
`final.ram` lists only changed bytes in access order. `final.ip` is the
architectural end-of-instruction value (the harness store-stub pad is
normalized out); for divide-trap cases it is the handler address
(IVT 0 -> 0000:0400, V20 convention — the 5-byte far-jump handler at
0x0400 is part of `initial.ram`).

Every odd-indexed case is **prefetched**: `initial.queue` holds the real
queue contents at the instruction's first-byte pop (6 bytes from even
addresses, 5 from odd — "as full as possible"), produced by a `63 C0`
preload run-up (verified side-effect-free on this silicon; the preload
bytes live below `cs:ip` and are not part of the record). Even-indexed
cases start from an empty queue. Consumers add `len(queue)` to the fetch
pointer as usual.

## Cycle rows (11 columns, 8086-suite shaped)

| Idx | Field | Notes |
|---|---|---|
| 0 | pin bitfield | bit0 = ALE, **synthesized** as T1 (no i8288 on the harness); INTR/NMI bits always 0 |
| 1 | multiplexed bus (20-bit) | T1 rows: address-phase sample (valid address); other rows: PS3-0 status in bits 19:16 + data-phase AD15-0 |
| 2 | segment status | decoded from PS1:PS0, valid on T2/T3/Tw of active cycles, else `--` |
| 3 | memory command string (`RAW`) | **synthesized** from bus status + T-state (8288-equivalent) |
| 4 | IO command string (`RAW`) | synthesized likewise |
| 5 | UBE̅ (address phase) | 0 = upper byte enabled; even addr + 0 = 16-bit transfer, odd addr + 0 = high-byte transfer, even + 1 = low-byte |
| 6 | data bus (16-bit) | data-phase AD15-0 sample |
| 7 | bus status | decoded BS2-0 **address-phase sample**. NOTE: the V30 asserts the next cycle's status during T4, so T4 rows show the upcoming cycle's type (V20 traces show PASV there) |
| 8 | T-state | Ti/T1/T2/T3/Tw/T4 (harness FSM annotation) |
| 9 | queue op | QS1:QS0 point sample: F/S/E/- |
| 10 | queue byte | from the shadow-queue reconstruction (fetch data pushed at T4, low byte first on even word fetches) |

Trace window: from the row carrying the test instruction's first-byte
pop (`F`) to the row carrying the next instruction's `F` (inclusive) —
for trapping cases, the handler's first byte. `initial.ram` contains the
instruction bytes, operand/stack/IVT bytes, and every byte actually
fetched or read up to the window end (0x90 fill and, late in the stream,
harness store-stub bytes — recorded truthfully; consumers replaying
`initial.ram` reproduce the cycle rows exactly).

`hash` = SHA1 of
`json.dumps([name,bytes,initial,final,cycles],separators=(",",":"))`.

## Known v0.1 limitations

- **No IN/port-read opcodes** (E4/E5/EC/ED, INM/6C-6D): the harness
  returns unconfigurable IOR data (suite convention is 0xFF); needs the
  planned RTL change.
- No segment-override / REP prefix randomization (prefix timing is
  characterized separately, docs/facts/measurements.md mission 10).
- 500 cases/opcode (8086-suite scale precedent), 26 opcodes.
- ALE and the 8288 command columns are synthesized (documented above);
  bus-status T4 rows differ from V20 shape (hardware-truthful).
- Flags in initial states never have TF set; reserved PSW bits read as
  the V30 forces them (15:12=1, bit1=1, bits 5,3=0).

## Generation conditions

uPD70116C-8, max (large) mode, CPU clock 4 MHz, zero wait states,
socketed on the nec_test DE10-Nano harness (`hdl/`), state
injection/extraction per `docs/notes/loadstore_design.md`, persistent
serve runner (`sw/v30run.py`). Emitted 2026-07-11/12.
